How to use Androguard to disassemble .class and .dex files

With Androlyze

Ok, if you would like to analyze an application with androguard, you must used androlyze to do that.

With the commande line

 ./androlyze.py -h
Usage: androlyze.py [options]

Options:
  -h, --help            show this help message and exit
  -i INPUT, --input=INPUT

  -m METHOD, --method=METHOD
                        display method(s) respect with a regexp
  -f FIELD, --field=FIELD
                        display field(s) respect with a regexp
  -s, --shell           open a shell to interact more easily with objects

  -x, --xpermissions    show paths of permissions

With the shell

The shell used in androlyze is ipython. So you must install it before.

With the shell, you can access to all created objects. For example to display methods of a dex file : 

pouik@camelot:~/androguard$ ./androlyze.py -s
Androlyze version ALPHA 0-update3
[~/androguard]
|2>j = DalvikVMFormat( open("./examples/android/Test/bin/classes.dex", "r").read() )
[~/androguard]
|3>for i in j.get_methods() :
   ...:     i.show()

or : 

pouik@camelot:~/androguard$ ./androlyze.py -s
Androlyze version ALPHA 0-update3
[~/androguard]
|2>j = DalvikVMFormat( open("./examples/android/Test/bin/classes.dex", "r").read() )
[~/androguard]
|3>x = VM_BCA( j )
[~/androguard]
|4>for i in j.get_methods() :
   ...:     i.pretty_show( x )

With the API

You can used the API to do the same things as androlyze. For example, you can open a dex file and show all methods (with pretty display which used the analysis module) : 

#!/usr/bin/env python                                                                                                                                                                                                             

import dvm, analysis, bytecode

TEST = "./examples/android/Test/bin/classes.dex"

j = dvm.DalvikVMFormat( open(TEST).read() )
x = analysis.VM_BCA( j )

# You can setup a specific display
bytecode.set_pretty_print(1)
#bytecode.set_pretty_print(0)


# SHOW METHODS
for i in j.get_methods() :
   i.pretty_show( x )

You can also have more details about the instruction : 

import androguard

TEST  = 'examples/java/test/orig/Test1.class'
#TEST = 'examples/android/Test/bin/classes.dex'

a = androguard.AndroguardS( TEST )

for method in a.get_methods() :
   print method.get_class_name(), method.get_name(), method.get_descriptor()
   code = method.get_code()
   bc = code.get_bc()

   idx = 0
   for i in bc.get() :
      print "\t", "%x" % idx, i.get_name(), i.get_operands()

      idx += i.get_length()

Now if you would like to see basic blocks of methods : 

a = androguard.AndroguardS( TEST )
x = analysis.VM_BCA( a.get_vm() )

for method in a.get_methods() :
   g = x.hmethods[ method ]

   g.basic_blocks.export_dot( OUTPUT + "%s-%s" % (method.get_name(), hashlib.md5( "%s-%s" % (method.get_class_name(), method.get_descriptor())).hexdigest()) + ".dot" )

   print method.get_class_name(), method.get_name(), method.get_descriptor()
   for i in g.basic_blocks.get() :
      print "\t %s %x %x" % (i.name, i.start, i.end)

With your own software : embedded androguard

By default, androguard is a standalone application, but it's possible to use it in your own application, for example to use only the disassembler : 

pouik@camelot:~/tmp/androguard$ mkdir myapps
pouik@camelot:~/tmp/androguard$ cp -r core/bytecode* myapps/
pouik@camelot:~/tmp/androguard$ cp core/misc.py myapps/
pouik@camelot:~/tmp/androguard$ cp -r core/analysis* myapps/
pouik@camelot:~/tmp/androguard/myapps$ cat test.py 
#!/usr/bin/env python

import dvm, analysis

TEST = "classes.dex"

j = dvm.DalvikVMFormat( open(TEST).read() )
x = analysis.VM_BCA( j )

# SHOW CLASSES (verbose and pretty) 
#j.pretty_show( x )

# SHOW METHODS
for i in j.get_methods() :
   i.pretty_show( x )
