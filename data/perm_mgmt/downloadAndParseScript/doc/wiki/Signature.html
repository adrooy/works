General

It's possible to create specific signature for a method in Androguard. The signature is based (but modified) on the paper of Silvio Cesare.

So a general signature has the following grammar : 

Procedure ::= StatementList
StatementList ::= Statement | Statement StatementList 
Statement ::= BasicBlock | Return | Goto | If | Field | Package | String | Exception
Return ::= 'R'
Goto ::= 'G'
If ::= 'I'
BasicBlock ::= 'B'
Field ::= 'F'0 | 'F'1
Package ::= 'P' PackageNew | 'P' PackageCall
PackageNew ::= 'C'
PackageCall ::= 'M'
PackageName ::= Epsilon | Id
String ::= 'S' Number | 'S' Id
Exception ::= Id
Number ::= \d+
Id ::= [a-zA-Z]\w+

So the signature is composed only with basic information about Control Flow Graph, Fields, Packages, Strings and Exceptions.

We have predefined signatures :

  • SIGNATURE_L0_0, no specific information about string, packages, fields (only S, P, F),
  • SIGNATURE_L0_1, idem L0_0 but with the size of strings (S+Size),
  • SIGNATURE_L0_2, idem L0_0 but filtering android packages. 
TEST_CASE  = 'examples/android/TC/bin/classes.dex'

a = androguard.AndroguardS( TEST_CASE )
x = analysis.VMAnalysis( a.get_vm() )

for method in a.get_methods() :
    print method.get_class_name(), method.get_name(), method.get_descriptor()
    print "-> : \t", x.get_method_signature(method, predef_sign = SIGNATURE_L0_0).get_string()
    print "-> : \t", x.get_method_signature(method, predef_sign = SIGNATURE_L0_1).get_string()
    print "-> : \t", x.get_method_signature(method, predef_sign = SIGNATURE_L0_2).get_string()
