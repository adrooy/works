Instructions

Basic instructions

The instructions of a dalvik virtual machine or the java virtual machine are represented into the dvm module or jvm module.

Even if the classes are different, you have the same methods in each class.

You can used the following methods :

  • get_name() : return the name of the instruction (string)
  • get_length() : return the length of the instruction (integer)
  • get_operands() : return a list of the operands
  • get_formatted_operands() : return a list of formatted operands (float, double, integer ...)
  • get_raw() : return the raw string which represent the instruction
  • show(pos) : display (stdout) the instruction (the first parameter must be an integer which represents the current position of the instruction into the method)
  • show_pos(pos) : exactly as show method but return a buffer

Examples

DVM

i.get_name(), i.get_operands(), i.get_formatted_operands()

--->

invoke-virtual/range [['v', 0], ['v', 1], ['v', 2], ['v', 3], ['v', 4], ['v', 5], ['v', 6], ['v', 7], ['v', 8], ['meth@', 36, 'Ltests/androguard/TestInvoke;', '(I I I I I I I I)', 'I', 'TestInvoke8']] []

const-wide/high16 [['v', 8], ['#+', 4607182418800017408L]] [('#d', 1.0)]

JVM

i.get_name(), i.get_operands(), i.get_formatted_operands()

--->

invokevirtual ['java/lang/StringBuilder', 'append', '(Ljava/lang/String;)Ljava/lang/StringBuilder;'] []
iconst_3 None []

Specific instructions

These instructions have more methods because they deal with more datas than classical instructions.

DVM

FillArrayData

  • format : namedtuple -> "<HHL" "ident element_width size"
  • get_data() : return the raw array

SparseSwitch

  • format : namedtuple -> "<HH" "ident size"
  • get_keys() : return the list of keys
  • get_targets() : return the list of targets (offsets)

PackedSwitch

  • format : namedtuple -> "<HHL" "ident size first_key"
  • get_targets() : return the list of targets (offsets)
